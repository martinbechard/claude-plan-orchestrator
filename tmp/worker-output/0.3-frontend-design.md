## 1. Problem Statement

The item detail page currently uses a flat two-column layout (left: traceability
chain of 5-6 stacked cards; right: plan tasks, artifacts, completions, traces).
Every card renders its full HTML content at page load. For items with many
artifacts, the page is long, slow to load, and hard to navigate.

The step explorer replaces the left column's flat card stack with a vertical
accordion of six pipeline stages. Each stage groups related artifacts (as
defined in D1), shows a status badge, and loads artifact content on-demand only
when the user expands that stage. The right column (plan tasks, completions,
traces) remains unchanged.

The item_detail() route adds one new key:

```python

```

### 2.3 Dynamic Endpoint Extension

The /item/{slug}/dynamic JSON response adds:

```json
{
    "stages": [
        {
            "id": "intake",
            "name": "Intake",
            "status": "done",
            "completed_at": "2026-03-29 14:23",
            "artifacts": [
                {"name": "User Request", "path": "tmp/plans/...", "timestamp": "..."},
            ]
        },
    ]
}
```

This allows live polling to update stage statuses and add new artifacts as they
appear during pipeline execution.

```html
          </time>
          <div class="artifact-content" aria-live="polite">
            <!-- Populated by AJAX on first expand -->
          </div>
        </li>

        <li class="artifact-item" data-artifact-path=".claude/workspace/.../clauses.md">
          <span class="artifact-name">Clause Register</span>
          <time class="artifact-timestamp" datetime="2026-03-29T13:15:00">
            Mar 29, 13:15
          </time>
          <div class="artifact-content" aria-live="polite"></div>
        </li>

      </ul>
    </div>
  </section>

  <!-- Additional stages follow the same pattern -->

</div>
```

### 3.2 Design Decisions on HTML

1. **button element for header**: Using a native button instead of a div+click
   ensures keyboard accessibility out of the box (focusable, Enter/Space toggle).
   aria-expanded and aria-controls link the header to its body for screen readers.

2. **section element for stage**: Provides a semantic grouping. The data-stage
   attribute is used by JS for localStorage key construction and dynamic updates.

3. **role="list" + role="listitem"**: Communicates the ordered sequence of stages
   to assistive technology. The stages render in pipeline order (D1: AC8).

4. **Separate artifact-content div**: Each artifact has a placeholder div that
   receives fetched content. This div is initially empty (AC24: no content at
   page load) and populated by AJAX on first expand (AC23, AC10).

5. **time elements with datetime**: Machine-readable timestamps for both stages
   and artifacts (AC3, AC4, AC13-AC15).

## 5. JavaScript: Accordion Behavior

### 5.1 Module Structure

All step explorer JS is encapsulated in a single IIFE appended to the existing
script block in item.html. No external JS files are needed.

```javascript
/* ── Step Explorer: accordion + lazy loading + localStorage persistence ── */
(function () {
    var STORAGE_PREFIX = 'step-explorer-';
    var slug = SLUG_VAR;  // from existing {{ slug | tojson }}
    var contentCache = new Map();
    var explorer = document.querySelector('.step-explorer');
    if (!explorer) return;

    var sections = explorer.querySelectorAll('.stage-section');

    /* ── Initialization ──────────────────────────────────────────────────── */

    initAccordion();

    function initAccordion() {
        sections.forEach(function (section) {
            var header = section.querySelector('.stage-header');
            var stageId = section.dataset.stage;

            header.addEventListener('click', function () {
                if (header.disabled) return;
                toggleStage(section);
            });

            // Keyboard: Enter/Space handled natively by <button>

            // Restore persisted state or apply default
            var persisted = getPersistedState(stageId);
            if (persisted === 'open') {
                expandStage(section);
            } else if (persisted === 'closed') {
                collapseStage(section);
            } else {
                // Default: expand the most recent active stage (D4)
                applyDefaultExpand(section);
            }
        });
    }

    /* ── Toggle / Expand / Collapse ──────────────────────────────────────── */

    function toggleStage(section) {
        var header = section.querySelector('.stage-header');
        var isExpanded = header.getAttribute('aria-expanded') === 'true';
        if (isExpanded) {
            collapseStage(section);
        } else {
            expandStage(section);
        }
    }

    function expandStage(section) {
        var header = section.querySelector('.stage-header');
        var body = section.querySelector('.stage-body');
        var stageId = section.dataset.stage;

        header.setAttribute('aria-expanded', 'true');
        body.hidden = false;
        persistState(stageId, 'open');

        // Trigger lazy loading for all artifacts in this stage
        loadArtifacts(section);
    }

    function collapseStage(section) {
        var header = section.querySelector('.stage-header');
        var body = section.querySelector('.stage-body');
        var stageId = section.dataset.stage;

        header.setAttribute('aria-expanded', 'false');
        body.hidden = true;
        persistState(stageId, 'closed');
    }

    /* ── Default expand logic ────────────────────────────────────────────── */

    function applyDefaultExpand(section) {
        // Find the most recent active (in_progress) stage
        // If none, expand the last "done" stage
        var activeSection = explorer.querySelector('.stage--in_progress');
        if (activeSection && activeSection === section) {
            expandStage(section);
            return;
        }
        if (activeSection) {
            // There IS an active stage, but this section is not it
            collapseStage(section);
            return;
        }
        // No active stage: expand the last done stage
        var doneSections = explorer.querySelectorAll('.stage--done');
        if (doneSections.length > 0) {
            var lastDone = doneSections[doneSections.length - 1];
            if (lastDone === section) {
                expandStage(section);
                return;
            }
        }
        collapseStage(section);
    }

    /* ── localStorage persistence ────────────────────────────────────────── */

    function storageKey(stageId) {
        return STORAGE_PREFIX + slug + '-' + stageId;
    }

    function getPersistedState(stageId) {
        try {
            return localStorage.getItem(storageKey(stageId));
        } catch (e) {
            return null;
        }
    }

    function persistState(stageId, state) {
        try {
            localStorage.setItem(storageKey(stageId), state);
        } catch (e) {
            // localStorage unavailable (private mode, quota) -- degrade silently
        }
    }

    /* ── AJAX content fetching with Map cache ────────────────────────────── */

    function loadArtifacts(section) {
        var items = section.querySelectorAll('.artifact-item');
        items.forEach(function (item) {
            var path = item.dataset.artifactPath;
            if (!path) return;

            var contentDiv = item.querySelector('.artifact-content');
            if (contentDiv.dataset.loaded) return; // already loaded

            // Check cache first
            if (contentCache.has(path)) {
                renderArtifactContent(contentDiv, contentCache.get(path), path);
                return;
            }

            // Show loading indicator
            contentDiv.innerHTML = '<div class="artifact-loading">' +
                'Loading content\u2026</div>';
            contentDiv.classList.add('artifact-content--loaded');

            var url = '/item/' + encodeURIComponent(slug) +
                '/artifact-content?path=' + encodeURIComponent(path);

            fetch(url)
                .then(function (r) {
                    if (!r.ok) throw new Error('HTTP ' + r.status);
                    return r.text();
                })
                .then(function (text) {
                    contentCache.set(path, text);
                    renderArtifactContent(contentDiv, text, path);
                })
                .catch(function (err) {
                    contentDiv.innerHTML = '<div class="artifact-content-inner">' +
                        'Failed to load: ' + escapeHtml(err.message) + '</div>';
                });
        });
    }

    function renderArtifactContent(contentDiv, text, path) {
        contentDiv.dataset.loaded = '1';

        // Determine if this is a markdown file (render as HTML) or plain text
        var isMd = path.endsWith('.md');
        if (isMd) {
            // For markdown, render using the server-side rendered version
            // Re-fetch as rendered HTML? No -- keep it simple: show as preformatted
            // text. The server-rendered HTML is the full page; inline rendering
            // would require a markdown-to-HTML JS library. Plain text display is
            // consistent and avoids adding dependencies.
            contentDiv.innerHTML = '<pre class="artifact-content-inner">' +
                escapeHtml(text) + '</pre>';
        } else {
            contentDiv.innerHTML = '<pre class="artifact-content-inner">' +
                escapeHtml(text) + '</pre>';
        }
        contentDiv.classList.add('artifact-content--loaded');
    }

    /* ── Artifact click-to-toggle ────────────────────────────────────────── */

    explorer.addEventListener('click', function (e) {
        var nameEl = e.target.closest('.artifact-name');
        if (!nameEl) return;

        var item = nameEl.closest('.artifact-item');
        var contentDiv = item.querySelector('.artifact-content');

        // Toggle visibility of already-loaded content
        if (contentDiv.classList.contains('artifact-content--loaded')) {
            contentDiv.classList.toggle('artifact-content--expanded');
            // When collapsing, set max-height to 0
            if (!contentDiv.classList.contains('artifact-content--expanded')) {
                contentDiv.style.maxHeight = '0';
            } else {
                contentDiv.style.maxHeight = '';
            }
        }
    });

    /* ── Expose for dynamic refresh integration ──────────────────────────── */

    window._stepExplorer = {
        updateStages: updateStagesFromDynamic,
        contentCache: contentCache,
    };

    /* ── Dynamic refresh: stage status updates ───────────────────────────── */

    function updateStagesFromDynamic(stages) {
        if (!stages || !stages.length) return;

        stages.forEach(function (stageData) {
            var section = explorer.querySelector(
                '[data-stage="' + stageData.id + '"]'
            );
            if (!section) return;

            // Update status class
            section.className = section.className
                .replace(/stage--\w+/, 'stage--' + stageData.status);

            // Update status icon
            var icon = section.querySelector('.stage-status-icon');
            if (icon) {
                icon.className = 'stage-status-icon stage-status--' + stageData.status;
                var statusLabel = stageData.status.replace(/_/g, ' ');
                icon.setAttribute('aria-label', 'Status: ' + statusLabel);
                icon.title = statusLabel.charAt(0).toUpperCase() + statusLabel.slice(1);
            }

            // Update artifact count
            var countEl = section.querySelector('.stage-artifact-count');
            if (countEl && stageData.artifacts) {
                var n = stageData.artifacts.length;
                if (n > 0) {
                    countEl.textContent = n + ' artifact' + (n !== 1 ? 's' : '');
                    countEl.classList.remove('stage-artifact-count--empty');
                }
            }

            // Update timestamp
            var tsEl = section.querySelector('.stage-timestamp');
            if (stageData.completed_at) {
                if (tsEl) {
                    tsEl.textContent = stageData.completed_at;
                } else {
                    var meta = section.querySelector('.stage-meta');
                    if (meta) {
                        var newTs = document.createElement('time');
                        newTs.className = 'stage-timestamp';
                        newTs.textContent = stageData.completed_at;
                        meta.appendChild(newTs);
                    }
                }
            }

            // Enable previously disabled button if artifacts appeared
            var header = section.querySelector('.stage-header');
            if (header.disabled && stageData.artifacts &&
                stageData.artifacts.length > 0) {
                header.disabled = false;
            }

            // Add new artifacts that were not in the initial render
            addNewArtifacts(section, stageData);
        });
    }

    function addNewArtifacts(section, stageData) {
        if (!stageData.artifacts) return;

        var list = section.querySelector('.artifact-list-stage');
        if (!list) {
            // Stage had no artifacts at page load; create the list
            var body = section.querySelector('.stage-body');
            var notice = body.querySelector('.stage-empty-notice');
            if (notice) notice.remove();
            list = document.createElement('ul');
            list.className = 'artifact-list-stage';
            list.setAttribute('aria-label', stageData.name + ' artifacts');
            body.appendChild(list);
        }

        var existingPaths = new Set();
        list.querySelectorAll('.artifact-item').forEach(function (item) {
            existingPaths.add(item.dataset.artifactPath);
        });

        stageData.artifacts.forEach(function (art) {
            if (existingPaths.has(art.path)) return;

            var li = document.createElement('li');
            li.className = 'artifact-item';
            li.dataset.artifactPath = art.path;

            var headerDiv = document.createElement('div');
            headerDiv.className = 'artifact-item-header';

            var nameSpan = document.createElement('span');
            nameSpan.className = 'artifact-name';
            nameSpan.textContent = art.name;
            headerDiv.appendChild(nameSpan);

            if (art.timestamp) {
                var tsEl = document.createElement('time');
                tsEl.className = 'artifact-timestamp';
                tsEl.textContent = art.timestamp;
                headerDiv.appendChild(tsEl);
            }

            li.appendChild(headerDiv);

            var contentDiv = document.createElement('div');
            contentDiv.className = 'artifact-content';
            contentDiv.setAttribute('aria-live', 'polite');
            li.appendChild(contentDiv);

            list.appendChild(li);
        });
    }

    /* ── Reuse existing escapeHtml from the polling script ────────────────── */
    // The escapeHtml function is already defined in the outer scope
    // of the existing polling IIFE. If scoping is an issue, duplicate:
    function escapeHtml(str) {
        var div = document.createElement('div');
        div.textContent = String(str);
        return div.innerHTML;
    }

}());
```

### 5.3 Content Cache Strategy

The JS Map cache (contentCache) follows these rules:

1. **Key**: File path string (same as data-artifact-path attribute)
2. **Value**: Raw text content from the artifact-content endpoint
3. **Lifetime**: Per page load (Map is reset on navigation/reload)
4. **No invalidation**: Artifact file content does not change during a single
   page session. If a file is modified by a worker, the user reloads the page
   to see changes (this is consistent with existing behavior).
5. **Cache hit on re-expand**: When a user collapses and re-expands a stage,
   artifacts are served from the Map instantly (AC10: avoids re-fetching).
6. **New artifacts from polling**: When the /dynamic endpoint reports new
   artifacts not yet in the DOM, addNewArtifacts() adds the DOM elements
   but does NOT pre-fetch their content. Content is fetched only when the
   user expands the stage (AC23: on-demand).

### 5.4 localStorage Key Design

Keys follow the pattern: step-explorer-{slug}-{stageId}

Examples:
- step-explorer-74-item-page-step-explorer-intake
- step-explorer-74-item-page-step-explorer-planning

Values: "open" or "closed"

This design:
- Scopes state to each item (via slug) so different items have independent
  accordion state.
- Scopes within an item to each stage (via stageId) so opening one stage
  does not affect others (AC5, AC6).
- Survives page reloads and browser restarts.
- Degrades gracefully in private browsing mode (try/catch around
  localStorage access).

## 8. Accessibility (WCAG 2.1 AA)

### 8.1 Keyboard Navigation

- **Tab**: Moves focus between stage headers (native button behavior).
- **Enter/Space**: Toggles expand/collapse of the focused stage (native button).
- **No custom key handlers needed**: Using native button elements means the
  browser handles keyboard interaction automatically.

### 8.2 Screen Reader Support

- **aria-expanded**: Announces whether each stage is currently open or closed.
- **aria-controls**: Links each header button to its corresponding body panel.
- **aria-label**: Provides descriptive labels for stages and their status.
- **aria-live="polite"**: On artifact-content divs, ensures screen readers
  announce newly loaded content without interrupting the user.
- **role="list" + role="listitem"**: Communicates the ordered sequence of stages.
- **Status announcements**: The status icon has an aria-label that spells out
  the status (e.g., "Status: in progress") so it is not conveyed solely by color.

- Stage name (#1a1a2e on #f8fafc): >15:1

- Status labels via aria-label (text, not visual)

## 13. Design Principles

1. **Progressive disclosure**: Stage headers show the minimum needed info
   (name, status, count, timestamp). Artifact content loads only on demand.
   This reduces cognitive load and page weight (AC24).

2. **Spatial consistency**: The accordion occupies the same left column as
   the current card stack. The right column is untouched. Users familiar
   with the current layout find the same information in the same place.

3. **State persistence**: localStorage remembers which stages were open,
   surviving page reloads. This respects the user's browsing context and
   prevents jarring layout shifts on revisit (D4).

4. **Zero dependencies**: No JS frameworks, no CSS preprocessors, no build
   step. The implementation uses the same vanilla JS + Jinja2 + inline CSS
   pattern as the rest of the codebase.

5. **Accessible by default**: Native button elements for headers, ARIA
   attributes for screen readers, reduced-motion media query for animation
   sensitivity. No custom keyboard handlers needed.

6. **Cache-friendly**: The JS Map cache prevents redundant network requests
   when users toggle stages repeatedly. New artifacts from polling are added
   to the DOM but not pre-fetched (respects on-demand principle).
