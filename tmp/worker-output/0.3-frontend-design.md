## 1. Problem Statement

The /execution-history/{run_id} page is an HTML shell that shows
"Loading execution tree..." but has NO JavaScript to fetch or render the tree.
The JSON API at /api/execution-tree/{run_id} already exists and returns a
nested tree of TreeNode objects.

This design specifies the concrete frontend implementation:
- execution-history.js file structure and fetch logic
- Recursive DOM builder for nested tree nodes
- CSS classes for indentation and node types
- Collapse/expand toggle with ARIA attributes
- Empty-tree fallback using run data from template context
- Loading spinner replacement
- Error handling for failed API calls
- Integration with base.html via extra_head block

### 1.1 Current State

The template (execution_history.html) extends base.html and contains:

```
<div id="execution-tree" data-run-id="{{ run_id }}">
  <p class="empty-state">Loading execution tree...</p>
</div>
```

The route (execution_history.py) already passes run_id and run (the root
trace row dict) to the template context. No JS file exists.

## 2. Data Contract: API Response

The /api/execution-tree/{run_id} endpoint returns:

```json
{
  "run_id": "abc-123",
  "tree": [
    {
      "run_id": "child-1",
      "name": "intake",
      "display_name": "intake",
      "node_type": "graph_node",
      "status": "success",
      "duration": 45.2,
      "cost": 0.0456,
      "model": "claude-sonnet-4-6",
      "token_count": 15000,
      "inputs_json": "...",
      "outputs_json": "...",
      "metadata_json": "...",
      "children": [...]
    }
  ]
}
```

Node types: "graph_node", "subgraph", "agent", "tool_call"
Statuses: "success", "error", "running", "unknown"

When the tree array is empty (no descendants), the page must show a
fallback using the root run data embedded in the template.

## 3. Template Changes: execution_history.html

### 3.1 Updated Template

```html
{% extends "base.html" %}

{% block title %}Execution History - {{ run_id[:8] }}{% endblock %}

{% block extra_head %}
<script src="/static/execution-history.js" defer></script>
{% endblock %}

{% block content %}
<h1>Execution History</h1>
<p>Run: <code>{{ run_id }}</code></p>

<div id="execution-tree"
     data-run-id="{{ run_id }}"
     data-run-name="{{ run.get('name', '') }}"
     data-run-status="{{ run.get('status', 'unknown') }}"
     data-run-start="{{ run.get('start_time', '') }}"
     data-run-end="{{ run.get('end_time', '') }}">
  <div class="exec-loading" role="status" aria-live="polite">
    <span class="exec-spinner" aria-hidden="true"></span>
    Loading execution tree&hellip;
  </div>
</div>
{% endblock %}
```

Design decisions:

1. **defer attribute on script**: The script runs after HTML parsing,
   so DOMContentLoaded is unnecessary. This matches the base.html inline
   script pattern (placed at end of body) but is more explicit.

2. **data-run-* attributes for fallback**: The run dict fields are embedded
   as data attributes so JS can render an informative empty-tree state
   without a second API call. Only essential fields are embedded (name,
   status, start, end) to keep the HTML lean.

3. **exec-loading with role="status"**: The loading indicator has ARIA
   live region semantics so screen readers announce when loading completes.

4. **exec- CSS prefix**: All new classes use "exec-" to avoid collisions
   with existing .detail-*, .badge-*, and .step-* classes in style.css.

### 3.2 Route Change: execution_history.py

The route already passes "run" (the trace row dict) to the template. The
"run" object has name, start_time, end_time, error, model, metadata_json
fields. No Python changes are needed -- the template accesses the existing
run dict via run.get() calls.

If run is None (404 case), the route already raises HTTPException before
reaching the template. No fallback needed at the route level.

## 4. JavaScript: execution-history.js

### 4.1 File Structure

```javascript
// langgraph_pipeline/web/static/execution-history.js
// Client-side tree fetch and render for /execution-history/{run_id}.
// Design: docs/plans/2026-04-02-83-trace-links-empty-or-missing-design.md (D4)

"use strict";

// -- Constants ---------------------------------------------------------------

var API_PREFIX = "/api/execution-tree/";
var INDENT_PX_PER_LEVEL = 20;
var CHEVRON_RIGHT = "\u25B6";  // right-pointing triangle
var CHEVRON_DOWN = "\u25BC";   // down-pointing triangle

// Node type labels for badges
var NODE_TYPE_LABELS = {
    graph_node: "Graph",
    subgraph: "Subgraph",
    agent: "Agent",
    tool_call: "Tool"
};

// -- Entry Point -------------------------------------------------------------

(function () {
    var container = document.getElementById("execution-tree");
    if (!container) return;

    var runId = container.dataset.runId;
    if (!runId) return;

    fetchAndRender(container, runId);
}());
```

Pattern notes: This follows the same "use strict" + IIFE pattern used by
analysis.js. Constants are at the top per the project's CLAUDE.md coding
conventions. The IIFE runs on script load (with defer, the DOM is ready).

### 4.2 Fetch and Render

```javascript
function fetchAndRender(container, runId) {
    fetch(API_PREFIX + encodeURIComponent(runId))
        .then(function (response) {
            if (!response.ok) {
                throw new Error("HTTP " + response.status);
            }
            return response.json();
        })
        .then(function (data) {
            container.innerHTML = "";
            if (!data.tree || data.tree.length === 0) {
                renderEmptyTree(container);
            } else {
                renderTree(container, data.tree);
            }
        })
        .catch(function (err) {
            renderError(container, err.message);
        });
}
```

Design decisions:
- **encodeURIComponent**: Defensive encoding of run_id in URL.
- **container.innerHTML = ""**: Replaces loading spinner with tree content.
- **Empty-tree detection**: data.tree.length === 0 triggers the fallback.
- **.catch()**: Any network or parse error renders an error message.

### 4.3 Recursive DOM Builder

```javascript
function renderTree(container, nodes) {
    var list = document.createElement("ul");
    list.className = "exec-tree";
    list.setAttribute("role", "tree");
    for (var i = 0; i < nodes.length; i++) {
        list.appendChild(buildNodeEl(nodes[i], 0));
    }
    container.appendChild(list);
}

function buildNodeEl(node, depth) {
    var li = document.createElement("li");
    li.className = "exec-node";
    li.setAttribute("role", "treeitem");
    li.style.paddingLeft = (depth * INDENT_PX_PER_LEVEL) + "px";

    var hasChildren = node.children && node.children.length > 0;
    var nodeId = "exec-node-" + node.run_id;

    // -- Header row --
    var header = document.createElement("div");
    header.className = "exec-node-header";

    // Expand/collapse toggle (only if has children)
    if (hasChildren) {
        var toggle = document.createElement("button");
        toggle.className = "exec-toggle";
        toggle.type = "button";
        toggle.setAttribute("aria-expanded", "true");
        toggle.setAttribute("aria-controls", nodeId + "-children");
        toggle.setAttribute("aria-label", "Toggle " + node.display_name);
        toggle.textContent = CHEVRON_DOWN;
        toggle.addEventListener("click", makeToggleHandler(toggle, nodeId));
        header.appendChild(toggle);
    } else {
        var spacer = document.createElement("span");
        spacer.className = "exec-toggle-spacer";
        header.appendChild(spacer);
    }

    // Node type badge
    var typeBadge = document.createElement("span");
    typeBadge.className = "exec-type exec-type-" + node.node_type;
    typeBadge.textContent = NODE_TYPE_LABELS[node.node_type] || node.node_type;
    header.appendChild(typeBadge);

    // Display name
    var name = document.createElement("span");
    name.className = "exec-name";
    name.textContent = node.display_name;
    header.appendChild(name);

    // Status badge
    var statusBadge = document.createElement("span");
    statusBadge.className = "badge badge-" + node.status;
    statusBadge.textContent = node.status;
    header.appendChild(statusBadge);

    // Metrics (duration, cost, tokens) -- only if nonzero
    if (node.duration > 0) {
        header.appendChild(makeMetric("exec-duration", fmtDuration(node.duration)));
    }
    if (node.cost > 0) {
        header.appendChild(makeMetric("exec-cost", fmtCost(node.cost)));
    }
    if (node.token_count > 0) {
        header.appendChild(makeMetric("exec-tokens", fmtTokens(node.token_count)));
    }
    if (node.model) {
        header.appendChild(makeMetric("exec-model", node.model));
    }

    li.appendChild(header);

    // -- Children container --
    if (hasChildren) {
        var childrenUl = document.createElement("ul");
        childrenUl.id = nodeId + "-children";
        childrenUl.className = "exec-children";
        childrenUl.setAttribute("role", "group");
        for (var j = 0; j < node.children.length; j++) {
            childrenUl.appendChild(buildNodeEl(node.children[j], depth + 1));
        }
        li.appendChild(childrenUl);
    }

    return li;
}
```

Design decisions:

1. **role="tree" / role="treeitem" / role="group"**: The WAI-ARIA tree
   pattern provides correct semantics for screen readers navigating a
   hierarchical tree view.

2. **Inline paddingLeft for depth**: Simpler than CSS calc() with
   attr(data-depth) which has limited browser support for calc values.
   The constant INDENT_PX_PER_LEVEL (20px) provides readable nesting.

3. **Children start expanded**: All nodes start expanded (aria-expanded=true)
   so the user sees the full tree on load. This is better for orientation
   than starting collapsed and requiring clicks to explore. For very deep
   trees, the user can collapse sections they do not need.

4. **Reuses existing .badge-success, .badge-error, .badge-unknown**: The
   status badge classes already exist in style.css. We add .badge-running
   for the "running" status.

5. **Conditional metric display**: Duration, cost, tokens, and model are
   only rendered when present (> 0 or non-empty). This avoids "$0.00" and
   "0 tokens" clutter for nodes without that data.

### 4.4 Toggle Handler (Collapse/Expand)

```javascript
function makeToggleHandler(toggleBtn, nodeId) {
    return function () {
        var childrenEl = document.getElementById(nodeId + "-children");
        if (!childrenEl) return;

        var isExpanded = toggleBtn.getAttribute("aria-expanded") === "true";
        toggleBtn.setAttribute("aria-expanded", String(!isExpanded));
        toggleBtn.textContent = isExpanded ? CHEVRON_RIGHT : CHEVRON_DOWN;
        childrenEl.hidden = isExpanded;
    };
}
```

Design decisions:
- **hidden attribute**: Uses the HTML hidden attribute (not CSS display:none)
  to remove children from both visual and accessibility trees when collapsed.
  This is the same pattern used by the step-explorer accordion.
- **aria-expanded toggle**: Screen readers announce the current expand state.
- **Closure over toggleBtn/nodeId**: Avoids stale references in loops.

### 4.5 Empty-Tree Fallback

```javascript
function renderEmptyTree(container) {
    var runName = container.dataset.runName || "Unknown";
    var runStatus = container.dataset.runStatus || "unknown";
    var startTime = container.dataset.runStart || "";
    var endTime = container.dataset.runEnd || "";

    var wrapper = document.createElement("div");
    wrapper.className = "exec-empty";

    var heading = document.createElement("h2");
    heading.textContent = "Run: " + runName;
    wrapper.appendChild(heading);

    var statusEl = document.createElement("p");
    var badge = document.createElement("span");
    badge.className = "badge badge-" + runStatus;
    badge.textContent = runStatus;
    statusEl.appendChild(document.createTextNode("Status: "));
    statusEl.appendChild(badge);
    wrapper.appendChild(statusEl);

    if (startTime) {
        var timeEl = document.createElement("p");
        timeEl.className = "exec-empty-meta";
        var duration = "";
        if (endTime && startTime) {
            var secs = (new Date(endTime) - new Date(startTime)) / 1000;
            if (secs > 0) duration = " (" + fmtDuration(secs) + ")";
        }
        timeEl.textContent = "Started: " + new Date(startTime).toLocaleString() + duration;
        wrapper.appendChild(timeEl);
    }

    var note = document.createElement("p");
    note.className = "exec-empty-note";
    note.textContent = "No detailed trace data available for this run. " +
        "This may be a synthetic trace created for a completion that ran " +
        "without LangSmith tracing enabled.";
    wrapper.appendChild(note);

    container.appendChild(wrapper);
}
```

This renders useful information when the API returns an empty tree array
(the root trace exists but has no child spans). The run metadata comes
from data-run-* attributes on the container div, which were populated
server-side from the run dict in the template context. No extra API call.

### 4.6 Error Handler

```javascript
function renderError(container, message) {
    container.innerHTML = "";
    var errDiv = document.createElement("div");
    errDiv.className = "exec-error";
    errDiv.setAttribute("role", "alert");

    var icon = document.createElement("span");
    icon.className = "exec-error-icon";
    icon.textContent = "!";
    icon.setAttribute("aria-hidden", "true");
    errDiv.appendChild(icon);

    var text = document.createElement("span");
    text.textContent = "Failed to load execution tree: " + message;
    errDiv.appendChild(text);

    container.appendChild(errDiv);
}
```

The role="alert" ensures screen readers announce the error immediately.
The error message includes the HTTP status or network error for debugging.

### 4.7 Formatting Helpers

```javascript
function fmtDuration(seconds) {
    var s = Math.round(seconds);
    if (s < 60) return s + "s";
    var m = Math.floor(s / 60);
    var rem = s % 60;
    if (m < 60) return m + "m " + rem + "s";
    var h = Math.floor(m / 60);
    return h + "h " + (m % 60) + "m";
}

function fmtCost(usd) {
    if (usd == null || usd === 0) return "";
    return "$" + usd.toFixed(4);
}

function fmtTokens(count) {
    if (!count) return "";
    if (count >= 1000) return (count / 1000).toFixed(1) + "k tok";
    return count + " tok";
}

function makeMetric(className, text) {
    var span = document.createElement("span");
    span.className = className;
    span.textContent = text;
    return span;
}
```

fmtDuration matches the pattern in dashboard.js (fmtElapsed). fmtCost
matches the project convention of plain "$0.0123" without tilde prefix.
fmtTokens uses "k tok" shorthand for readability.

### 5.2 Design Decisions on CSS

1. **exec- prefix**: All classes use "exec-" to namespace them away from
   existing CSS. The codebase already has .step-*, .badge-*, .detail-*,
   .json-block, etc. The prefix prevents any collision.

2. **Flexbox for node rows**: .exec-node-header uses display:flex with
   gap:0.5rem. This matches the header patterns in .detail-header and
   provides natural wrapping and alignment of badge + name + metrics.

3. **Border-left tree guide**: .exec-node has a 2px left border (#e2e8f0)
   that creates a visual tree guide line connecting parent to children.
   This is the same border color used throughout style.css.

4. **Node type colors**: Each type gets a distinct background/foreground
   pair that meets WCAG AA contrast. Colors align with the existing
   palette: blue for graph nodes, purple for subgraphs, amber for agents,
   green for tool calls.

5. **Reuses .badge base class for status**: The JS generates
   class="badge badge-success" etc., reusing existing badge styles.
   Only .badge-running is new (amber background matching in_progress).

6. **Reduced motion**: The loading spinner respects prefers-reduced-motion
   by stopping the animation and showing a static indicator.

## 6. Integration with base.html

The script tag in the extra_head block:

```html
{% block extra_head %}
<script src="/static/execution-history.js" defer></script>
{% endblock %}
```

This is the standard pattern. The defer attribute ensures:
1. Script downloads in parallel with HTML parsing
2. Script executes after the DOM is ready
3. No need for DOMContentLoaded listener inside the JS

The script is page-specific (only loaded on execution-history pages),
not added to base.html. This follows the same pattern as dashboard.js
(loaded only on /dashboard) and analysis.js (loaded only on /analysis).

## 7. Accessibility

### 7.1 ARIA Tree Pattern

The tree uses the WAI-ARIA tree view pattern:

| Element | Role/Attribute | Purpose |
|---|---|---|
| Outer ul | role="tree" | Identifies the widget as a tree |
| Each li | role="treeitem" | Each node is a tree item |
| Children ul | role="group" | Groups children under their parent |
| Toggle button | aria-expanded | Communicates expand/collapse state |
| Toggle button | aria-controls | Links to children container id |
| Toggle button | aria-label | "Toggle {display_name}" for context |
| Loading div | role="status", aria-live="polite" | Announces loading completion |
| Error div | role="alert" | Announces errors immediately |

### 7.2 Keyboard Navigation

| Key | Context | Action |
|---|---|---|
| Tab | Anywhere | Moves focus between toggle buttons |
| Enter/Space | Toggle button focused | Expand or collapse children |
| Escape | Expanded node | Could be extended to collapse |

The toggle buttons are native button elements, so Tab, Enter, and Space
work without any additional keyboard event handling.

### 7.3 Hidden Content

The hidden attribute on children containers (set by the toggle handler)
removes content from both the visual layout and the accessibility tree.
This prevents screen readers from reading collapsed subtrees.

## 8. Interaction Flows

### 8.1 Normal Load (Tree Has Children)

1. Page loads, "Loading execution tree..." with spinner shown
2. JS reads data-run-id from container
3. JS fetches /api/execution-tree/{run_id}
4. API returns tree with children
5. JS clears container, calls renderTree()
6. Full tree rendered expanded (all children visible)
7. User clicks toggle buttons to collapse subtrees they are not interested in

### 8.2 Empty Tree (No Descendants)

1. Page loads with spinner
2. JS fetches API -- response has tree: []
3. JS clears container, calls renderEmptyTree()
4. Fallback renders: run name, status badge, start time + duration, and
   an explanatory note about synthetic traces
5. Data comes from data-run-* attributes (no extra API call)

### 8.3 API Error

1. Page loads with spinner
2. JS fetches API -- network error or non-200 response
3. JS clears container, calls renderError()
4. Error message displayed with red icon and HTTP status
5. role="alert" causes screen readers to announce the error

### 8.4 User Collapses/Expands Nodes

1. User clicks the chevron toggle on a node with children
2. aria-expanded flips to "false", chevron changes to right-pointing
3. Children ul gets hidden attribute set
4. Clicking again: aria-expanded flips to "true", hidden removed

| AC | How Satisfied | Implementation Detail |

| AC2 | Tree displays content | renderTree() builds recursive DOM from API response; replaces "Loading..." placeholder |
