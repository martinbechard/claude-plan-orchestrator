## 1. Problem Statement

The item detail page currently renders all artifact content inline at page load,
creating a slow, long page. The step explorer accordion (already scaffolded in
item.html) groups artifacts under six pipeline stages. This design specifies:

- On-demand artifact content loading via AJAX (AC7, AC8, AC27)
- Loading spinner while content fetches (AC28)
- Collapsible stage sections with independent toggle (AC12, AC13, AC14)
- Per-artifact timestamps (AC25, AC26)
- localStorage persistence for collapse state
- /dynamic polling integration for live stage status updates

### 1.1 Current Layout (Already Implemented)

The two-column grid is already in place:

```

```

## 2. Data Contract: Backend to Template

### 2.1 StageInfo (from build_stages() in item.py)

The template receives a "stages" list. Each entry is a StageInfo TypedDict:

```python
StageInfo = {
    "id": "intake",                       # lowercase identifier
    "name": "Intake",                     # display label
    "status": "done",                     # "not_started" | "in_progress" | "done"
    "artifacts": [ArtifactInfo, ...],     # ordered list
    "completion_ts": "2026-04-02 14:23",  # formatted string or None
    "completion_epoch": 1743609780.0,     # raw float or None
}
```

### 2.2 ArtifactInfo (per artifact within a stage)

```python
ArtifactInfo = {
    "name": "User Request",              # display label (AC29: not "Raw Input")
    "path": "tmp/plans/.claimed/74-item-page-step-explorer.md",  # for AJAX fetch
    "timestamp": 1743605400.0,           # raw epoch float (AC25, AC26)
    "timestamp_display": "2026-04-02 13:10",  # pre-formatted string
}
```

Key: timestamp is the file st_mtime (AC26), pre-formatted server-side as
"YYYY-MM-DD HH:MM" in timestamp_display (AC25).

### 2.3 /dynamic Endpoint Stage Summary

The /item/{slug}/dynamic response includes lightweight stage summaries:

```json
{
  "stages": [
    {
      "id": "intake",
      "status": "done",
      "completion_ts": "2026-04-02 14:23",
      "completion_epoch": 1743609780.0,
      "artifact_count": 3
    }
  ]
}
```

This omits artifact details (full ArtifactInfo) to keep polling responses small.
The JS polling handler uses artifact_count changes to invalidate cached content.

## 3. HTML Structure (Jinja2 Template)

### 3.1 Stage Section Anatomy

Each stage renders as a div (not section) with a button header and body region.
The "collapsed" CSS class on .step-stage controls visibility. All stages start
collapsed; JS expands them based on saved state or default heuristic.

```html
<div class="step-explorer" data-slug="{{ slug }}">
  {% for stage in stages %}
  <div class="step-stage collapsed"
       data-stage-id="{{ stage.id }}"
       data-stage-status="{{ stage.status }}"
       data-artifact-count="{{ stage.artifacts | length }}">

    <button class="step-stage-header"
            type="button"
            aria-expanded="false"
            aria-controls="step-body-{{ stage.id }}">
      <span class="step-stage-chevron" aria-hidden="true">&#9654;</span>
      <span class="step-stage-name">{{ stage.name }}</span>
      <span class="badge step-status-badge step-status-{{ stage.status }}">
        {{ stage.status | replace('_', ' ') }}
      </span>
      {% if stage.completion_ts %}
      <span class="step-stage-ts">{{ stage.completion_ts }}</span>
      {% endif %}
      <span class="step-stage-count">
        {{ stage.artifacts | length }}
        artifact{{ 's' if stage.artifacts | length != 1 else '' }}
      </span>
    </button>

    <div class="step-stage-body" id="step-body-{{ stage.id }}" role="region"
         aria-label="{{ stage.name }} stage artifacts">
      {% if stage.artifacts %}
        {% for artifact in stage.artifacts %}
        <div class="step-artifact">
          <div class="step-artifact-meta">
            <span class="step-artifact-name">{{ artifact.name }}</span>
            {% if artifact.timestamp_display %}
            <span class="step-artifact-ts">{{ artifact.timestamp_display }}</span>
            {% endif %}
          </div>
          <pre class="step-artifact-content"
               data-artifact-path="{{ artifact.path }}"
               hidden>Loading&hellip;</pre>
        </div>
        {% endfor %}
      {% else %}
        <div class="step-empty">No artifacts yet</div>
      {% endif %}
    </div>
  </div>
  {% endfor %}
</div>
```

### 3.2 Design Decisions on HTML

1. **button element for header (AC12, AC13)**: Native button provides keyboard
   accessibility (Tab, Enter, Space) without extra JS. aria-expanded and
   aria-controls link header to body for screen readers.

2. **data-stage-id attribute**: Used as the localStorage key suffix and as the
   CSS selector for /dynamic polling updates.

3. **data-stage-status and data-artifact-count**: Data attributes on .step-stage
   store current server state. The /dynamic handler compares artifact_count to
   detect new artifacts and invalidate cached content.

4. **pre element with data-artifact-path**: Each artifact has a hidden pre that
   receives fetched content. The "hidden" attribute keeps it invisible until the
   stage is expanded and content is loaded. Using pre preserves whitespace for
   log files and code.

5. **All stages start collapsed**: The "collapsed" class is set server-side.
   JavaScript removes it based on localStorage or default heuristic (expand
   in_progress stages on first visit). This avoids a flash of expanded content
   before JS runs.

6. **Chevron as text entity (&#9654;)**: A right-pointing triangle rotated 90
   degrees via CSS transform when expanded. No icon font or SVG dependency.

### 3.3 Empty/Not-Started Stage

Stages with no artifacts render a "No artifacts yet" message in the body.
The header button is still clickable (not disabled) so users can check.
The status badge shows "not started" in grey.

### 3.4 Per-Artifact Timestamp Display (AC25, AC26)

Each artifact displays its timestamp_display value in a monospace span:

```html
<span class="step-artifact-ts">2026-04-02 13:10</span>
```

The timestamp is the file's st_mtime, computed server-side (AC26). It appears
next to the artifact name in the same row, providing at-a-glance temporal
context without cluttering the layout.

## 5. JavaScript Behavior

### 5.1 Collapse/Expand with localStorage (AC12, AC13, AC14)

```javascript
(function () {
    var slug = SLUG_FROM_TEMPLATE;  // {{ slug | tojson }}
    var STORAGE_PREFIX = 'step-explorer-' + slug + '-';

    function loadStageArtifacts(stageEl) {
        stageEl.querySelectorAll('.step-artifact-content').forEach(function (pre) {
            if (pre.dataset.loaded) { return; }
            pre.hidden = false;
            pre.classList.add('loading');
            pre.textContent = '';
            pre.dataset.loaded = '1';
            var url = '/item/' + encodeURIComponent(slug) +
                '/artifact-content?path=' + encodeURIComponent(pre.dataset.artifactPath);
            fetch(url)
                .then(function (r) {
                    if (!r.ok) { throw new Error('HTTP ' + r.status); }
                    return r.text();
                })
                .then(function (t) {
                    pre.classList.remove('loading');
                    pre.textContent = t;
                })
                .catch(function (err) {
                    pre.classList.remove('loading');
                    pre.textContent = 'Failed to load: ' + err.message;
                });
        });
    }

    document.querySelectorAll('.step-stage').forEach(function (stage) {
        var stageId = stage.dataset.stageId;
        var header = stage.querySelector('.step-stage-header');
        var saved = localStorage.getItem(STORAGE_PREFIX + stageId);

        // Default: expand in_progress stages on first visit
        var shouldExpand;
        if (saved !== null) {
            shouldExpand = saved === 'expanded';
        } else {
            shouldExpand = stage.dataset.stageStatus === 'in_progress';
        }

        if (shouldExpand) {
            stage.classList.remove('collapsed');
            header.setAttribute('aria-expanded', 'true');
            loadStageArtifacts(stage);
        }

        header.addEventListener('click', function () {
            var isCollapsed = stage.classList.toggle('collapsed');
            header.setAttribute('aria-expanded', String(!isCollapsed));
            localStorage.setItem(STORAGE_PREFIX + stageId,
                isCollapsed ? 'collapsed' : 'expanded');
            if (!isCollapsed) { loadStageArtifacts(stage); }
        });
    });
```

Key design choices:

- **Each stage toggles independently (AC14)**: The click handler only affects
  the clicked stage's classList. No "close all others" logic.
- **localStorage keyed by slug + stageId**: Persists across page reloads.
  Different work items have independent state.
- **Default expand heuristic**: On first visit (no saved state), in_progress
  stages are expanded. This surfaces the most relevant content without
  overwhelming the user.
- **data-loaded caching**: Once fetched, artifacts are not re-fetched on
  collapse/re-expand. The data-loaded attribute acts as a simple cache flag.

### 5.2 On-Demand Content Loading (AC7, AC8, AC27)

Content is fetched only when a stage is expanded (not at page load). The
loadStageArtifacts() function:

1. Iterates over all .step-artifact-content pre elements in the stage
2. Skips any with data-loaded attribute (already fetched)
3. Shows the pre (hidden=false), adds .loading class for spinner (AC28)
4. Fetches via /item/{slug}/artifact-content?path={artifact.path}
5. On success: removes .loading, sets textContent
6. On error: removes .loading, shows error message

This satisfies AC7 (artifacts not loaded at page load), AC8 (faster initial
load), and AC27 (content loaded when user expands).

### 5.3 Loading Indicator (AC28)

The .loading CSS class on the pre element:
- Changes background to light grey (#f8fafc)
- Shows a CSS spinner via ::before pseudo-element
- Displays "Loading..." text (or empty, with just the spinner)
- Is removed when fetch completes (success or error)

This provides clear visual feedback that content is being fetched, satisfying
AC28.

### 5.4 /dynamic Polling Integration

The existing selective refresh polls /item/{slug}/dynamic every 10 seconds.
The step explorer exposes a global function for the polling handler to call:

```javascript
    window._stepExplorer = {
        updateStages: function (stages) {
            for (var i = 0; i < stages.length; i++) {
                var s = stages[i];
                var stageEl = document.querySelector(
                    '.step-stage[data-stage-id="' + s.id + '"]');
                if (!stageEl) { continue; }

                var oldCount = parseInt(stageEl.dataset.artifactCount || '0', 10);
                stageEl.dataset.stageStatus = s.status;
                stageEl.dataset.artifactCount = String(s.artifact_count);

                // Update status badge text and class
                var badge = stageEl.querySelector('.step-status-badge');
                if (badge) {
                    badge.className = 'badge step-status-badge step-status-' + s.status;
                    badge.textContent = s.status.replace(/_/g, ' ');
                }

                // Update or create completion timestamp
                var tsEl = stageEl.querySelector('.step-stage-ts');
                if (s.completion_ts) {
                    if (!tsEl) {
                        tsEl = document.createElement('span');
                        tsEl.className = 'step-stage-ts';
                        var countEl = stageEl.querySelector('.step-stage-count');
                        if (countEl) {
                            countEl.parentNode.insertBefore(tsEl, countEl);
                        }
                    }
                    tsEl.textContent = s.completion_ts;
                } else if (tsEl) {
                    tsEl.textContent = '';
                }

                // Update artifact count label
                var countEl2 = stageEl.querySelector('.step-stage-count');
                if (countEl2) {
                    countEl2.textContent = s.artifact_count +
                        ' artifact' + (s.artifact_count !== 1 ? 's' : '');
                }

                // Invalidate cache when new artifacts appear
                if (s.artifact_count > oldCount) {
                    stageEl.querySelectorAll(
                        '.step-artifact-content[data-loaded]'
                    ).forEach(function (pre) {
                        delete pre.dataset.loaded;
                    });
                }
            }
        }
    };
}());
```

The polling handler in the refresh IIFE calls:

```javascript
if (data.stages && window._stepExplorer) {
    window._stepExplorer.updateStages(data.stages);
}
```

Design choices for live updates:

- **Badge class swap**: Directly replaces the CSS class to update badge color.
- **Timestamp creation on the fly**: If a stage transitions to "done" during
  polling, a timestamp span is created and inserted before the count span.
- **Cache invalidation on artifact_count increase**: When the server reports
  more artifacts than the DOM's data-artifact-count, all data-loaded flags
  are cleared so the next expand triggers fresh fetches. This handles the
  case where new artifacts appear while the page is open.
- **No full DOM rebuild**: The handler updates attributes and text in place
  rather than re-rendering stages. This preserves expand/collapse state and
  avoids layout shifts.

## 6. Interaction Flows

### 6.1 First Visit (No Saved State)

1. Page loads with all stages collapsed (server renders "collapsed" class)
2. JS reads localStorage -- no saved state for this slug
3. JS expands stages with status "in_progress" (removes "collapsed", sets
   aria-expanded="true")
4. For each expanded stage, loadStageArtifacts() fetches content via AJAX
5. Spinner shown during fetch (AC28); content displayed when complete

### 6.2 User Expands a Collapsed Stage (AC13)

1. User clicks stage header button (or presses Enter/Space)
2. JS toggles "collapsed" class off
3. aria-expanded set to "true"
4. localStorage updated: "expanded"
5. loadStageArtifacts() called -- fetches any unfetched artifacts
6. Chevron rotates to down-pointing via CSS transition

### 6.3 User Collapses an Expanded Stage (AC12)

1. User clicks stage header button
2. JS toggles "collapsed" class on
3. aria-expanded set to "false"
4. localStorage updated: "collapsed"
5. Content remains in DOM (cached); no re-fetch needed on re-expand
6. Chevron rotates back to right-pointing

### 6.4 Live Polling Detects New Artifact

1. /dynamic returns stages with higher artifact_count for a stage
2. updateStages() updates data-artifact-count attribute
3. Clears data-loaded on all artifact pre elements in that stage
4. Next time the user expands (or if already expanded), fresh content loads
5. Note: the HTML for new artifacts is not added dynamically -- the user must
   reload the page to see new artifact rows. The cache invalidation ensures
   existing artifact content is refreshed.

### 6.5 Return Visit (Saved State Exists)

1. Page loads with all stages collapsed
2. JS reads localStorage and finds saved preferences
3. Stages saved as "expanded" are expanded; "collapsed" stay collapsed
4. Default heuristic (expand in_progress) is overridden by saved state

## 7. Accessibility

### 7.1 Keyboard Navigation

- Tab moves focus between stage header buttons
- Enter/Space toggles collapse/expand (native button behavior)
- No custom key handlers needed

### 7.2 ARIA Attributes

- aria-expanded on button: "true"/"false" reflects current state
- aria-controls on button: points to step-body-{stageId} region
- role="region" on body div with aria-label for stage name
- Status badge text is readable by screen readers ("not started", "in progress",
  "done")

### 7.3 Focus Management

- focus-visible outline (2px #3b6fcf) on header buttons
- No focus trap -- users can Tab through all stage headers freely
- Expanded content is in the natural tab order below its header
